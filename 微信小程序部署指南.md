# 微信小程序部署指南

本指南帮助你从零把「AI 英语绘本生成器」部署到微信小程序，并确保后端与小程序都能正常使用。

---

## 一、本地先跑通（必做）

### 1. 安装依赖

```bash
cd /Users/jianhua/Downloads/ai-英语绘本生成器
npm install
```

### 2. 配置 Gemini API Key

在项目根目录创建 `.env.local`（若已有则编辑），填入你的 Gemini API Key：

```bash
GEMINI_API_KEY=你的Gemini_API_Key
```

（Key 可在 [Google AI Studio](https://aistudio.google.com/apikey) 申请。）

### 3. 启动后端

```bash
npm run server
```

看到 `API server running at http://localhost:3001` 即表示后端已启动。

### 4. 测试网页版（可选）

- 不设 `VITE_API_URL` 时，网页会直连 Gemini（Key 会暴露在前端，仅适合本地/测试）。
- 若希望网页也走后端，在 `.env.local` 增加：

```bash
VITE_API_URL=http://localhost:3001
```

然后执行：

```bash
npm run dev
```

浏览器打开 http://localhost:3000，试一次「生成英语故事」和「点击单词翻译」，确认可用。

### 5. 用微信开发者工具跑小程序

1. 打开 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)，用微信扫码登录。
2. 选择「小程序」→「导入项目」。
3. 目录选择本项目的 **`miniprogram`** 文件夹（不要选整个项目根目录）。
4. AppID：  
   - 若已注册小程序，选你的 AppID。  
   - 若没有，选「测试号」或「使用测试号」即可。
5. 在开发者工具里勾选：**「详情」→「本地设置」→「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**。  
   这样本地可用 `http://localhost:3001` 请求后端。
6. 确认 `miniprogram/config.js` 里为：

   ```js
   baseUrl: 'http://localhost:3001',
   ```

7. 保持本机 `npm run server` 在运行，在模拟器里点「创作」→ 选年级 →「生成英语故事」，应能生成并进入阅读；点单词可翻译、加入生词本。  
   若失败，看控制台报错和 Network 里请求是否发到 `http://localhost:3001`。

## 二、部署后端（上线前必做）

小程序正式版**不能**用 `localhost`，必须用 **HTTPS** 且已备案的域名。请先部署后端再改小程序配置。

### 方式 A：云开发 / 云函数（推荐新手）

- 在微信公众平台使用「云开发」，把「生成故事」「翻译单词」做成云函数，在云函数里调用 Gemini API（需在云环境配置 `GEMINI_API_KEY`）。
- 云函数 URL 会类似：`https://xxx.service.tcloudbase.com/...`，后续在小程序里请求该地址即可。

### 方式 B：自有服务器（Node 部署）

1. 准备一台有公网 IP 的服务器（或使用腾讯云、阿里云等），并有一个 **已备案** 的域名，例如 `api.yourdomain.com`。
2. 在服务器上安装 Node.js（建议 18+），把本项目的 **根目录**（包含 `server/`、`package.json`、`.env.local` 等）上传或 git clone 到服务器。
3. 在服务器上：

   ```bash
   cd 项目根目录
   npm install
   # 确保 .env.local 里有 GEMINI_API_KEY
   node server/index.js
   ```

4. 用 Nginx（或 Caddy）做反向代理，把 `https://api.yourdomain.com` 指向本机 `http://127.0.0.1:3001`，并配置 **HTTPS**（可用 Let’s Encrypt 免费证书）。
5. 测试：在浏览器或 Postman 请求：

   ```text
   POST https://api.yourdomain.com/api/generate-story
   Content-Type: application/json
   Body: {"grade":"Grade 6"}
   ```

   能返回 JSON 即表示后端部署成功。

### 方式 C：Vercel / Railway / 其他 Node 托管

- 将本项目部署为 Node 服务，把「启动命令」设为 `node server/index.js`，在环境变量里配置 `GEMINI_API_KEY`。
- 部署后会得到一个 HTTPS 地址，例如 `https://your-app.vercel.app`，后续在小程序里用这个地址作为 API 根地址。

## 三、配置小程序请求域名（正式/体验版必做）

1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → 你的小程序。
2. 进入 **开发管理** → **开发设置** → **服务器域名**。
3. 在 **request 合法域名** 中点击「修改」。
4. 填入你的后端 **HTTPS 域名**（不要带路径），例如：
   - `https://api.yourdomain.com`
   - 或云开发/云函数的请求域名。
5. 保存。注意：域名需已备案，且每月修改次数有限制。

---

## 四、改小程序里的 API 地址

在 **`miniprogram/config.js`** 中，把 `baseUrl` 改成你已部署好的 **HTTPS 根地址**（不要以 `/` 结尾）：

```js
module.exports = {
  baseUrl: 'https://api.yourdomain.com',  // 或你的云函数/托管地址
};
```

保存后，在微信开发者工具中重新编译、预览或上传。

---

## 五、上传与发布

1. 在微信开发者工具中，确认：
   - 已关闭「不校验合法域名…」（或使用体验版/正式版再测一遍）。
   - `miniprogram/config.js` 的 `baseUrl` 已改为正式 HTTPS 域名。
2. 点击右上角 **「上传」**，填写版本号与备注，上传代码。
3. 登录微信公众平台 → **版本管理** → 找到刚上传的版本，设为 **体验版** 或提交 **审核** 发布为正式版。

## 六、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 本地小程序请求失败 | 未勾选「不校验合法域名」或后端未启动 | 勾选不校验、确认 `npm run server` 在跑、`config.js` 为 `http://localhost:3001` |
| 正式/体验版请求失败 | 未配置 request 合法域名或不是 HTTPS | 在公众平台配置 request 域名为你的 HTTPS 后端地址 |
| 生成故事/翻译报错 | 后端未配置或未读到 `GEMINI_API_KEY` | 在服务器/云函数的环境变量中设置 `GEMINI_API_KEY` |
| 小程序打开白屏 | 页面报错或路径错误 | 在开发者工具控制台看报错，检查 `app.json` 的 pages 与文件名是否一致 |

## 七、项目结构简要说明

- **网页端**：`src/` 下 React 应用，可用 `VITE_API_URL` 指向后端，避免在前端暴露 API Key。
- **后端**：`server/index.js` + `server/geminiHandlers.js`，提供 `/api/generate-story`、`/api/translate-word`，供网页与小程序共用。
- **小程序**：`miniprogram/` 为完整小程序目录，直接作为「导入项目」的目录；通过 `config.js` 的 `baseUrl` 访问后端。

按上述步骤做完「本地跑通 → 部署后端 → 配置域名 → 改 baseUrl → 上传」，即可完成从本地到微信小程序的完整部署。
