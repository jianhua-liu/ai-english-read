# 境外后端部署说明（解决云函数 60 秒超时）

腾讯云函数在国内访问 Google Gemini 容易超时。把 API 部署到**境外**（如 Vercel）后，小程序改请求该地址即可。

---

## 一、部署到 Vercel

### 1. 准备

- 注册 [Vercel](https://vercel.com)（可用 GitHub 登录）。
- 本机安装 [Node.js](https://nodejs.org)。
- 项目根目录已有 `api/` 文件夹（`api/generate-story.js`、`api/translate-word.js`、`api/lib/gemini.js`）。

### 2. 部署

**方式 A：用 Vercel 网页（推荐）**

1. 把本项目推送到 **GitHub**（若还没有）：  
   - 在 GitHub 新建仓库，把当前项目 push 上去。
2. 打开 [vercel.com](https://vercel.com) → **Add New** → **Project**。
3. **Import** 你的 GitHub 仓库，**Root Directory** 保持为仓库根目录（不要选子目录）。
4. 在 **Environment Variables** 里添加：
   - **Name**：`GEMINI_API_KEY`  
   - **Value**：你的 Gemini API Key  
5. 点 **Deploy**，等待完成。
6. 部署成功后记下域名，形如：`https://你的项目名.vercel.app`。

**方式 B：用 Vercel CLI**

```bash
cd /Users/jianhua/Downloads/ai-英语绘本生成器
npm i -g vercel
vercel
```

按提示登录、关联项目，在 Vercel 网站里为该项目设置环境变量 `GEMINI_API_KEY`，然后重新部署一次。

### 3. 确认接口可用

在浏览器或 Postman 测试：

- URL：`https://你的项目名.vercel.app/api/generate-story`  
- 方法：**POST**  
- Body（JSON）：`{"grade":"Grade 6"}`  

若返回一段 JSON（含 `title`、`pages`），说明部署成功。

**说明**：Vercel 免费版单次请求约 10 秒超时；若生成故事仍超时，可考虑 [Vercel Pro](https://vercel.com/pricing)（60 秒）或改用 [Railway](https://railway.app) 等部署同一套 `api/` 逻辑。

## 二、小程序改用境外后端

1. 打开 **miniprogram/config.js**。
2. 将 `baseUrl` 改为你的 Vercel 地址（**不要**在末尾加 `/`）：
   ```js
   baseUrl: 'https://你的项目名.vercel.app',
   ```
3. **微信公众平台** → 你的小程序 → **开发管理** → **开发设置** → **服务器域名** → **request 合法域名** → 添加：`https://你的项目名.vercel.app`（仅域名，不要路径）。
4. 保存后，在微信开发者工具里**重新编译**，再点「生成英语故事」测试。

---

## 三、行为说明

- **config.baseUrl 为空**：小程序仍走**微信云函数**（国内可能 60 秒超时）。
- **config.baseUrl 填了境外地址**：小程序请求该地址的 `/api/generate-story` 和 `/api/translate-word`，由境外服务器调用 Gemini，一般不再超时。

按上述步骤部署并配置后，生成故事应能正常完成。若某一步报错，把报错信息或截图发出来即可继续排查。
